## Specs as built
- **Trim size:** 6" × 9" (432 × 648 pt) — exact page size, no crop marks.
- **Page count:** **293 pages** (build r7, with full front + back matter).
  IngramSpark needs an **even** count for perfect binding — add one trailing
  blank page → **294** before upload (see ingramspark-spec.md §1).
- **Margins (symmetric, gutter-safe):** sides 0.75", top 0.75", bottom 0.70".
  - IngramSpark minimum is 0.5" from trim; recommended gutter for ≤300 pp is
    ~0.625". 0.75" both sides clears both with margin. (Symmetric margins are
    used instead of mirrored inside/outside — simpler and fully compliant.)
- **Body:** IBM Plex Serif 11/15.5, justified, 16 pt first-line indent
  (no indent on first paragraph of a chapter or after a scene break).
- **Scene breaks:** centered `* * *`.
- **Fonts:** IBM Plex Serif Regular/Italic/Bold all **embedded (subset)**.
  Verified: zero non-embedded / base-14 fonts (the default Helvetica slot was
  overridden to an embedded TTF).
- **Page numbers:** centered footer; body numbering starts at **1 on Chapter
  One**. The 6 front-matter pages are unnumbered.
- **Text color:** pure **K-only black** (`CMYKColor(0,0,0,1)`) on body, headings,
  page numbers, and all front/back matter — converts to 100% K, 0 CMY.
- **Widows/orphans:** body and front/back-matter paragraph styles set
  `allowWidows=0, allowOrphans=0`; chapter number + title carry `keepWithNext=1`
  so a heading can never strand at the foot of a page. Residual limitation:
  reportlab's widow/orphan handling is best-effort and works per-paragraph — it
  cannot guarantee a fully balanced last line in justified text, and it does not
  do TeX-grade page-breaking. A human visual proof pass is still advised; any
  single stray short line can be nudged by minor wording in editing (not done
  here — chapters are frozen).
- **No bleed** — correct for a text-only interior (nothing bleeds off-trim).

## Caveat to resolve before final upload (not possible in this environment)
- **Color space / PDF standard:** this PDF is **RGB**, not **PDF/X-1a:2001
  (CMYK + output intent)**, which is IngramSpark's *preferred* format. Text is
  pure black (0,0,0). IngramSpark accepts standard PDFs and will preflight/
  convert, but for a guaranteed pass either:
  1. let IngramSpark's uploader convert it, or
  2. run it through Acrobat Pro "Save as PDF/X-1a" or Ghostscript
     (`gs -dPDFX -dProcessColorModel=/DeviceCMYK ...` with a PDF/X def file).
  Ghostscript/Acrobat are not available here, so that final conversion is a
  one-step manual finish on a machine that has them.

## UPDATE (2026-06-21): PDF/X-1a CMYK now produced in-repo

Ghostscript was installed in the build environment, so the CMYK / PDF/X-1a:2001
conversion is **no longer a manual step** — it is generated by `tools/make_pdfx.sh`.

- **Print deliverable:** `Saeren-Chronicles-Book-One-6x9-interior-<REV>-PDFX1a.pdf`
  (current: `...-r7-PDFX1a.pdf`).
- **Verified:** 294 pages (even, binding-ready), color space **DeviceCMYK** (zero
  DeviceRGB), `GTS_PDFX` OutputIntent present, all fonts embedded, text 100% K.
- **Pipeline:** `python3 tools/assemble_manuscript.py` → `python3 tools/build_pdf.py`
  (now adds full front/back matter + forces an even page count) → `bash tools/make_pdfx.sh`.
- **ICC caveat:** the OutputIntent uses Ghostscript's generic `default_cmyk.icc`.
  This passes PDF/X-1a preflight, but for best press match swap in a US SWOP
  (or your printer's specified) profile by editing the path in `tools/PDFX_def.ps`.
- IngramSpark also accepts the RGB file and will auto-convert; the PDF/X-1a file
  is the "guaranteed clean preflight" option.
