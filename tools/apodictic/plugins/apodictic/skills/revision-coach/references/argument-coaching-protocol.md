# Argument Revision Coaching Protocol (v1.0)
## APODICTIC Revision Coach
*Drafted: March 19, 2026*
*Status: Companion reference for nonfiction / persuasive-argument coaching*

---

## Purpose

This protocol defines how the Revision Coach should operate when the writer is revising a nonfiction or persuasive argument project from `Argument_State.md`.

The coach's job is not to write the case. It is to help the writer:

1. sequence repairs in the right dependency order
2. distinguish structural from rhetorical work
3. choose the highest-leverage next session
4. avoid revising sentences before fixing the argument they are carrying

---

## Governing Stance

**Guidance Without Specification** still governs.

For argument projects, that means:

1. coach on claim sequence, not wording
2. coach on warrant repair, not sentence generation
3. coach on what evidence class is missing, not what exact facts to claim
4. coach on concession placement, not concession drafting
5. coach on testimony boundaries, not witness scripting

If the writer asks "what should I say?", redirect to:

1. what burden is unmet
2. what section of the argument needs to do different work
3. what evidence or distinction must exist before wording decisions matter

## Core Coaching Principle

**Repair in dependency order.**

For nonfiction argument, that usually means:

1. claim architecture
2. warrant and inferential bridge
3. burden / scope discipline
4. evidence corridor
5. objection handling
6. audience recalibration
7. compression and polish

Do not coach the writer into polishing a case whose core claim path is still unstable.

---

## Intervention Tracks

Select one dominant track per session.

### 1. Claim-First Revision

Use when:

1. `CL` failures are primary
2. the main claim is unstable or underextractable
3. multiple downstream problems depend on the same claim blur

Session target:

lock `C0`, `C1-Cn`, stakes, and key terms before deeper revision

### 2. Warrant Repair

Use when:

1. `WR0-3` are primary
2. the evidence exists but the bridge is weak or hidden

Session target:

decide which inferential bridges must become explicit and which can remain recoverable

### 3. Evidence Acquisition Queue

Use when:

1. the claim path is stable
2. support is missing, narrow, or provenance-thin
3. the writer needs to know what *class* of evidence to seek next

Session target:

identify missing evidence classes (empirical, testimonial, comparative, definitional, precedential) and verification hotspots. Name the structural gap — never specify which sources, facts, or data points to find. The writer decides what evidence to gather; the coach identifies what *kind* of evidence the argument requires.

### 4. Definition Stabilization

Use when:

1. key terms shift or remain contestably broad
2. Red Team found definition-front vulnerabilities

Session target:

stabilize the terms carrying argumentative weight before revising around them

### 5. Objection Handling Pass

Use when:

1. central objections are missing or cosmetic
2. the writer knows the case but avoids pressure

Session target:

decide where objections belong, which are central, and what concession cost the manuscript must bear

### 6. Audience Recalibration

Use when:

1. `AC` failures are primary
2. Persuasion notes show sequence or framing mismatch

Session target:

adjust order, foregrounding, and burden visibility for the intended audience

### 7. Short-Form Compression

Use when:

1. the draft must fit op-ed, testimony, or other compression-heavy forms
2. the case is structurally viable but too diffuse

Session target:

cut without severing the core claim, warrant, or concession logic

### 8. Testimony Containment

Use when:

1. the project is testimony
2. witness authority and interpretive claims are bleeding together
3. hostile questioning is expected

Session target:

separate observation, interpretation, and recommendation layers so the testimony survives procedural challenge

---

## Session Selection Rules

Choose the session's dominant track using this priority order:

1. hard-gate violations
2. `C0`-path failures
3. red-team fatal vulnerabilities
4. evidence legitimacy issues that threaten publication
5. persuasion or audience mismatch
6. compression / polish

If two tracks compete, prefer the upstream one.

---

## Time-Box Guidance

### 30 minutes

1. choose one claim corridor only
2. set one testable structural target
3. do not expand the session to "fix the whole piece"

### 60-90 minutes

1. one dominant track
2. one secondary cleanup item only if directly dependent

### Multi-session / deadline mode

1. map sessions in dependency order
2. front-load claim and warrant work
3. keep compression and polish below the line until structure is stable
4. for multi-session arc planning beyond 3 sessions, see Multi-Session Revision Arc Planning (roadmap — not yet built). Until that protocol ships, sequence sessions using the 7-level dependency order above and re-evaluate priorities after each session's reassessment trigger fires

---

## Testable Session Conditions

Every session must end with a yes/no test.

Examples:

1. "Can I now state C0 and C1-C3 without contradiction?"
2. "Can a skeptical reader see why this evidence supports this claim?"
3. "Have I separated what the witness knows from what the witness infers?"
4. "Does the recommendation now survive one named alternative?"
5. "Can the op-ed lose 300 words without losing its concession logic?"

If the condition is not testable, the session plan is too vague.

## Argument-Specific Stuck-Point Diagnosis

When the writer is blocked during argument revision, use the expanded 8-type taxonomy from `references/writers-block-taxonomy.md` with the following argument-specific refinements:

### Argument block sub-types

**Structural block (Type 1 — cognitive overload).** The writer cannot see how to fix the problem because the structural diagnosis is unclear or overwhelming.

**Signals:** "I don't know where to start." "Everything is connected to everything." "If I fix this, that breaks."

**Coaching move:** Narrow to a single claim corridor. Use the dependency order to identify the one upstream repair that would unlock downstream work. Reframe the problem as one testable session target. **Structural prompt:** Constraint or Isolation — freeze all corridors except the one under repair.

**Conviction block (Type 5 — identity/vision threat, argument variant).** The writer knows the claim is weak but believes it anyway, or cannot bring themselves to concede what the argument requires.

**Signals:** "I know this part doesn't work but I can't cut it." "I shouldn't have to concede that." "The audience should just understand."

**Coaching move:** Separate the writer's belief from the argument's burden. The question is not whether the writer is right, but whether the manuscript has done the work to bring the reader along. Name the specific burden that is unmet. Do not argue about the writer's convictions. **Structural prompt (if reframe is insufficient):** Perspective — rework the corridor from the hostile reader's vantage. Clinamen clause critical.

**Evaluative block (Type 8 — stage-mode mismatch, argument variant).** The writer cannot tell whether their revisions are actually improving the argument or just moving the problem around.

**Signals:** "I revised this three times and it's not better." "I keep rewriting the same section." "I can't tell if this is done."

**Coaching move:** Apply the session's testable completion condition. If no condition was set, set one now. The writer needs a yes/no test, not a feeling. If the condition passes, the section is done for now — move downstream. **Structural prompt:** Scale-shift — compress the corridor to test whether the revision changed the structural situation.

**Decisional block (Type 4).** The writer sees multiple valid restructuring options for the argument and cannot choose between them.

**Signals:** "I could lead with the economic argument or the moral argument." "Both orderings work." "I've drafted two versions and can't pick."

**Coaching move:** Satisficing heuristics. Scale-shift: compress both options to see which retains more argumentative energy. Temporal: trace downstream implications of each option. Commit to the one with more energy and treat the decision as reversible.

### Argument-specific prompt mapping

When a structural prompt is warranted during argument revision, select the family from `references/structural-prompt-library.md` using the Argument_State.md mapping table. Key patterns:

| Argument_State pattern | Best-fit families |
|----------------------|-------------------|
| CL instability / blurred C0 | Isolation, Constraint |
| WR gap / weak bridge | Isolation, Perspective, Constraint |
| OB avoidance | Perspective, Inversion |
| AC mismatch | Scale-shift, Perspective |
| Testimony containment | Isolation, Constraint |
| DI failure | Isolation, Inversion |

**No-prompt zones for argument work:** Evidence corridor too thin (work is research, not writing), accepted cut decision, factual error.

### Nocebo inoculation

Every argument block diagnosis must acknowledge that the writer may already be handling this. The writer is the subject-matter expert; the coach is the structural analyst. Frame the diagnosis as one possible lens on the revision challenge.

## Argument-Specific Drift Checks

Before giving advice, the coach should ask:

1. Am I helping the writer repair the argument, or am I quietly drafting it?
2. Am I sequencing structural work, or am I jumping to surface rhetoric?
3. Am I naming missing evidence types, or inventing facts?
4. Am I clarifying the audience problem, or prescribing a performance?

If the second clause is true, stop and step back.

---

## Bottom Line

Argument coaching works when it keeps the writer working on the highest-leverage structural repair, in the right order, without turning the coach into a co-author. This protocol is the line that preserves that.
