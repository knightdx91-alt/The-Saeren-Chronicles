# Session Plan and Revision Calendar Templates

*Output format reference for the APODICTIC Revision Coach.*

---

## Session Plan

Use for Session Planning and Momentum Tracking modes.

```markdown
# Session Plan — [Date]

**Project:** [Project name]
**Available time:** [stated time]
**Energy level:** [structural / mixed / polish]
**Diagnostic basis:** [run label of the DE synthesis being coached from]

---

## Focus: [Root cause or issue name]

**Why this first:** [One sentence — what this unlocks, what depends on it, why it has the highest leverage]

**Scope:** [Specific scenes, chapters, or sections — not the whole manuscript]

**What it needs:** [Intervention class — structural requirement, not specific content. Use the language from the DE synthesis.]

**What "done" looks like:** [Testable condition. The writer should be able to answer yes/no to this after revising.]

**Estimated time:** [Minutes or hours — honest, not optimistic]

---

## Structural Experiment (optional — included when coach diagnoses a block amenable to prompting)

**Block diagnosis:** [type + refinement, e.g., "Cognitive overload — too many interdependent variables in the Act 2 restructuring"]

**Prompt family:** [e.g., Isolation]

**Prompt:**
[Full prompt text, satisfying all 5 firewall conditions, including clinamen clause and self-check]

**Framing:** This is low-stakes exploration, not new canonical draft material. The exercise is disposable. If it moves you, keep it. If it doesn't, the information about what didn't work is the value.

**Note:** You may already be handling this structural problem through your own process. This prompt is one possible way in, not the only one.

---

## If Time Remains: [Second priority]

**Why this second:** [One sentence]

**Scope:** [Specific scenes/chapters]

**What it needs:** [Intervention class]

**What "done" looks like:** [Testable condition]

**Estimated time:** [Minutes or hours]

---

## Do NOT Touch Today

- [Issue that looks tempting but is lower leverage or depends on today's work]
- [Issue that requires structural energy when the writer is in polish mode]
- [Issue that will be affected by today's focus — revise this AFTER today's work settles]

---

## Reassessment Trigger

[What to check after completing the Focus item. Examples:]
- "After restructuring Act 2, re-run Pass 1 (Reader Experience) on the affected chapters to check whether pacing issues resolved or migrated."
- "No reassessment needed — this was an isolated scene fix."

---

## Session Notes

*Fill in during or after the session:*

**Completed:**

**Deferred:**

**New issues surfaced:**

**Next session priority:**
```

```markdown
| Issue | Severity | Why Deferred | Impact of Deferral |
|-------|----------|--------------|--------------------|
| | | | |

---

## Buffer

[10-20% of total hours reserved for unexpected complications, new issues surfacing from revisions, and rest. Do not schedule this time. If unused, apply to highest-leverage deferred item.]

---

## Weekly Check-In Protocol

At the start of each week:
1. Review prior week's completed/deferred items
2. Adjust remaining schedule if prior week's work shifted priorities
3. Note any new issues that surfaced
4. Confirm next week's focus still has the highest leverage

---

```
