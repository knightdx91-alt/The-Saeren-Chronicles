# Specialized Audit: Plot Structure Selection & Coaching
## Version 1.1
*Last Updated: March 2026*

---

## Purpose

This audit works **upstream** of the Plot Architecture audit. Where Plot Architecture diagnoses whether a spine is functioning correctly in a completed or drafted manuscript, this audit helps writers **choose, combine, and adapt** structural tools before or during drafting — or when a draft isn't working and needs a structural rethink.

**When to activate:**
- Pre-drafting: Author has material but hasn't committed to a structure
- Stuck draft: Author knows something's wrong but can't name it
- Structural pivot: Existing spine isn't serving the material
- Hybrid design: Author wants to layer multiple structures deliberately

**Core principle:** Structure should emerge from what the story *needs to do to its reader*, not from template loyalty. This audit helps match story concerns to structural tools, then checks whether the chosen combination is coherent.

---

## Phase 1: Story Concern Mapping

Before selecting a structure, identify what the story needs to accomplish. Stories are driven by one or more **core concerns** — the fundamental thing the narrative must deliver.

### The Concern Inventory

Ask the author (or derive from manuscript/notes):

**1. What must the reader FEEL?**

| If the answer is... | Primary structural family |
|---------------------|--------------------------|
| Mounting dread | Rhythm & Intensity (Wave/Pulse, Lullaby, Pressure Cooker) |
| Intellectual satisfaction | Information & Knowledge (Mystery, Puzzle Box, Conspiracy) |
| Emotional devastation | Linear & Teleological (Freytag, Fichtean) |
| Trapped / suffocated | Constraint & Environment (Siege, Countdown, Pressure Cooker, Procedural) |
| Disturbed by own complicity | Relationship & Erotic (Seduction, Betrayal of Self, Training) |
| Transformed alongside character | Existential & Identity (Hero's Journey, Heroine's Journey, Bildungsroman) |
| Uncertain what's real | Information & Knowledge (Rashomon) + Tonal (Psychological Horror) |
| Catharsis | Moral & Social (Justice/Revenge, Redemption) |
| Contemplative recognition | Linear & Teleological (Kishōtenketsu) — meaning from juxtaposition, not collision |

**2. What is the story's ENGINE — the thing that generates forward motion?**

| Engine type | Candidate spines |
|-------------|-----------------|
| A question (who did it? what's happening?) | Mystery, Conspiracy, Revelatory, Puzzle Box |
| A desire (I want / I need) | Story Circle, Courtship, Faustian, Quest |
| A constraint (I can't escape) | Captivity, Siege, Countdown, Pressure Cooker, Procedural |
| A pattern (this keeps happening) | Spiral, Fugue, Loop, Wave/Pulse |
| A relationship (what's between us) | Courtship, Seduction, Two-Handed Causality, Training |
| A transformation (who am I becoming?) | Corruption, Bildungsroman, Heroine's Journey, Metamorphosis |
| A moral reckoning (what have I done / what was done to me?) | Aftermath, Redemption, Scapegoat, Justice/Revenge |
| A clock (time is running out) | Countdown, Fichtean, Seven-Point |
| A juxtaposition (placing things side by side) | Kishōtenketsu — meaning from contrast, not collision |

**3. What is the story's RELATIONSHIP TO TRUTH?**

| Truth relationship | Structural implications |
|-------------------|----------------------|
| Truth is discoverable | Mystery, Investigation — spine must deliver resolution |
| Truth is constructed | Rashomon, Clinical Case File — spine must sustain ambiguity |
| Truth is retrospective | Revelatory, Nonlinear Reveal, Reverse Chronology — spine must produce retroactive meaning |
| Truth is eroding | Psychological Horror, Spiral — spine must destabilize certainty |
| Truth is contested | Braided, Two-Handed — spine must balance competing accounts |

## Phase 2: Spine Selection Protocol

### For Single-Spine Stories

Most stories have ONE primary spine. Use this decision tree:

```
START: What generates forward motion?
│
├─ A QUESTION → Does the question have an answer?
│  ├─ Yes → Mystery or Investigation
│  ├─ No, but it deepens → Conspiracy or Revelatory
│  └─ No, and that's the point → Psychological Horror
│
├─ A DESIRE → Does the character get what they want?
│  ├─ Yes, at a cost → Faustian or Story Circle
│  ├─ No, and they transform → Heroine's Journey or Bildungsroman
│  └─ They get it and it destroys them → Corruption
│
├─ A CONSTRAINT → Does the constraint tighten?
│  ├─ Yes, steadily → Pressure Cooker
│  ├─ Yes, in crises → Fichtean Curve
│  ├─ Yes, with a deadline → Countdown
│  └─ Yes, but character finds micro-agency → Captivity
│
├─ A PATTERN → Does the pattern escalate or deepen?
│  ├─ Escalates (same thing, worse) → Spiral
│  ├─ Deepens (same thing, new meaning) → Fugue
│  ├─ Oscillates (crests and troughs) → Wave/Pulse
│  └─ Repeats with variation → Loop/Groundhog
│
├─ A RELATIONSHIP → What's at stake in the relationship?
│  ├─ Intimacy as risk → Courtship
│  ├─ Power as desire → Seduction
│  ├─ Conditioning as horror → Training
│  ├─ Mutual escalation → Two-Handed Causality
│  └─ Self-betrayal through desire → Betrayal of Self
│
├─ A TRANSFORMATION → What kind?
│  ├─ Moral fall → Corruption
│  ├─ Identity formation → Bildungsroman
│  ├─ Integration of split self → Heroine's Journey
│  ├─ Physical/ontological → Metamorphosis
│  └─ Becoming-other → Doppelgänger
│
└─ A JUXTAPOSITION → Does meaning come from placing things side by side?
   ├─ Yes, and no conflict needed → Kishōtenketsu
   └─ Yes, but conflict also present → Check if Kishōtenketsu or
      Three-Act with thematic contrast
```

### For Multi-Spine Stories

Most manuscripts layer 2-3 structural tools. The key question: **which spine governs, and which modifies?**

**Primary spine:** Determines the macro-shape of the narrative. If this spine fails, the book fails.

**Secondary spine(s):** Add texture, rhythm, or a parallel engine. If these fail, the book is weaker but still functional.

**Structural technique overlays:** Modify presentation without governing shape (see Phase 3).

**Example combinations:**

| Story description | Primary | Secondary | Technique overlay |
|------------------|---------|-----------|-------------------|
| Conditioning horror with escalating intimacy | Pressure Cooker | Training + Seduction | Talisman (recurring object) |
| Investigation that recontextualizes the past | Mystery | Revelatory | Nested Dolls (layered memory) |
| Dual POV romance with countdown | Courtship | Countdown | TV/Serial (alternating threads) |
| Literary novel about compulsion and identity | Spiral | Heroine's Journey | Fugue (motif repetition) |
| Institutional horror with procedural structure | Procedural | Conspiracy | Clinical Case File (observation vs. reality) |
| Erotic horror novella about trust and violation | Lullaby | Betrayal of Self | Talisman (object's meaning shifts) |

## Phase 3: Structural Technique Overlays

These are not spines — they don't govern narrative shape. They are **presentation techniques** that modify how a spine delivers its payload.

### TV/Serial Format

**What it does:** Organizes material into teaser → A/B/C threads → tag. Each "episode" (chapter or section) runs parallel storylines with scheduled beats.

**When to apply:**
- Multiple characters with independent but thematically linked arcs
- Work that benefits from cross-cutting between threads
- Any manuscript where one thread's revelation should land while another thread's tension is suspended

**Logic Gate: The Thread Complement**
```
CHECK: Do A/B/C threads illuminate each other thematically?

IF cutting from Thread A to Thread B produces irony or resonance → PASS
IF threads are thematically independent → FLAG: "Parallel Tracks"
IF one thread consistently undermines another's tension → FLAG: "Tension Leak"
```

**Compatible with:** Braided Narrative, Two-Handed Causality, Siege
**Fights with:** Spiral (needs single-thread focus), Lullaby (needs unbroken rhythm)

---

### Game-Inspired / Choice-Driven Format

**What it does:** Structures the narrative as a series of trials that produce transformation. Each "level" introduces a constraint, tests the character, and yields power or trauma.

**When to apply:**
- Conditioning narratives where each phase changes the character
- Stories with escalating tests or challenges
- Training arcs where the character accumulates skills/damage

**Logic Gate: The Level Differentiation**
```
CHECK: Does each "level" demand a DIFFERENT response from the character?

IF each trial requires new strategy/skill/sacrifice → PASS
IF trials are variations on the same test → FLAG: "Grinding, Not Progressing"
IF final "boss" doesn't require synthesis of earlier learning → FLAG: "Disconnected Capstone"
```

**Compatible with:** Training, Pressure Cooker, Fichtean Curve
**Fights with:** Wave/Pulse (game structure is linear; wave is oscillating)

---

### Jo-ha-kyū (Pacing Overlay)

**What it does:** Imposes a three-phase rhythmic shape — measured opening (Jo), accelerating complexity (Ha), rapid culmination (Kyū) — at any scale: scene, act, or whole work. Not a spine; a pacing lens that diagnoses rhythmic flatness within any identified spine.

**When to apply:**
- Manuscript feels metronomic despite sound structure
- Pacing feels "same speed throughout"
- Individual scenes lack internal rhythmic shape
- Work needs rhythmic contour independent of plot logic

**Note:** Traditional proportions are not equal thirds. Jo is roughly one-fifth, Ha three-fifths, Kyū one-fifth. The rhythm is fractal — it should operate at multiple scales simultaneously.

**Logic Gate: The Rhythm Check**
```
CHECK: Does the manuscript accelerate across its phases?

IF Jo (measured) → Ha (faster, denser) → Kyū (fastest) → PASS
IF pacing is constant throughout → FLAG: "Metronomic — no rhythmic shape"
IF every scene follows identical acceleration → FLAG: "Rhythmic Monotony"
```

**Compatible with:** All spines (it's an overlay, not a spine)
**Fights with:** Pressure Cooker (resists Jo's measured opening); rigid application fights itself (predictable acceleration becomes its own flatness)

## Phase 4: Hybrid Structure Design

For stories that need to operate at multiple scales or combine incompatible-seeming structures.

### The Layer Model

Complex narratives operate structures at different scales simultaneously:

| Scale | What it governs | Example structures |
|-------|----------------|-------------------|
| **Micro** (scene) | Individual scene mechanics | Scene Turn logic, Lullaby rhythm |
| **Meso** (chapter/section) | Section-level arcs | Episodic, TV/Serial format, Seven-Point |
| **Macro** (whole work) | Overall narrative shape | Three-Act, Freytag, Hero's Journey, Corruption |
| **Meta** (series) | Multi-work arc | See Fantasy & Series Architecture supplement |

**The design principle:** Structures at different scales rarely conflict. A Spiral (macro) can contain Lullaby scenes (micro) organized in Episodic chapters (meso) within a series using Expanding Quest architecture (meta).

**Structures conflict when they operate at the SAME scale with incompatible assumptions:**
- Lullaby + Fichtean at micro level (one needs comfort; the other eliminates it)
- Mystery + Psychological Horror at macro level (one promises answers; the other promises ambiguity)

### Designing a Hybrid

**Step 1:** Identify the scales your story needs to operate at.

**Step 2:** For each scale, select the structural tool that serves your core concern at that level.

**Step 3:** Check for same-scale conflicts using the Compatibility Matrix (Plot Architecture audit).

**Step 4:** Identify the **handoff points** — moments where one structure yields to another.

**Handoff types:**
- **Clean transition:** One structure completes, another begins (e.g., Corruption Arc → Redemption Arc)
- **Nesting:** One structure contains another (e.g., Nested Dolls frame containing a Mystery spine)
- **Oscillation:** Structures alternate (e.g., Wave crests use Fichtean escalation; troughs use Fugue reflection)
- **Scale separation:** Structures coexist at different scales without interfering (e.g., Spiral macro + Lullaby micro)

### Hybrid Design Template

```
STORY: [title/description]

CORE CONCERN: [what must the story do to the reader?]

STRUCTURAL LAYERS:
  Micro (scene): [technique/structure]
  Meso (section): [technique/structure]
  Macro (whole): [primary spine]
  Meta (series): [if applicable]

SECONDARY ENGINES: [structures that add texture without governing shape]

TECHNIQUE OVERLAYS: [presentation formats]

POTENTIAL CONFLICTS: [same-scale tensions to monitor]

HANDOFF POINTS: [where structures transition or yield]
```

## Phase 5: Structural Triage (For Stuck Drafts)

When a draft isn't working and the author can't identify why, use this diagnostic sequence.

### Step 1: Name the Symptom

| What the author says | What it usually means |
|---------------------|----------------------|
| "It's boring" | Engine failure — the thing generating forward motion has stalled |
| "It's repetitive" | Recursion without variation — needs Fugue rules (each return must cost or reveal) |
| "It's too neat" | Single-spine limitation — needs secondary engine or recursive element |
| "It meanders" | Missing teleology — needs linear beats (Save the Cat, Seven-Point) |
| "It's all middle" | Proportional failure — Act II bloat; check Three-Act percentages |
| "Something's off but I can't name it" | Spine mismatch — the structure isn't serving the story's core concern |
| "Every diagnostic flags problems but the draft works" | Wrong structural paradigm — check for Kishōtenketsu before assuming conflict-driven spine |
| "It's intense but flat" | Missing oscillation — needs Wave/Pulse or Lullaby rhythm |
| "Characters are interesting but plot is dead" | Character engine, no plot spine — needs external structure (Countdown, Mystery, Quest) |
| "Plot works but characters are thin" | Plot spine without identity engine — needs Heroine's Journey, Bildungsroman, or Betrayal of Self |
| "The reveal doesn't land" | Missing retroactive infrastructure — needs Revelatory Plot setup or Nested Dolls |
| "It's just one damn thing after another" | Crisis without metabolization — apply Fichtean recovery ratio check |

### Step 2: Identify Current Spine

Run Plot Architecture audit's Step 1 (spine identification) on the existing draft. The spine the manuscript is USING may not be the spine the author INTENDED.

### Step 3: Compare Current vs. Needed

| | Current spine | Core concern demands |
|-|---------------|---------------------|
| **Engine** | [what's generating motion now] | [what should generate motion] |
| **Reader experience** | [what reader currently feels] | [what reader should feel] |
| **Resolution type** | [what the current spine promises] | [what the story needs to deliver] |

### Step 4: Prescribe

Three options:

**A. Strengthen the current spine.** Run Plot Architecture logic gates; fix what's broken. Sometimes the right spine is installed but malfunctioning.

**B. Swap the primary spine.** The material needs a different governing structure. This usually means significant revision but clarifies what scenes need to do.

**C. Add a secondary engine.** The primary spine is correct but insufficient. Layer a secondary structure to provide what's missing (rhythm, constraint, identity arc, etc.).

## Integration with Core Framework

This audit operates at three points in the workflow:

1. **Pre-intake (for new projects):** Before the core framework runs, help author choose structure. Output feeds into Contract.
2. **During synthesis (for stuck drafts):** When Plot Architecture diagnoses structural failure, this audit prescribes alternatives.
3. **During revision planning:** When an author is considering structural overhaul, this audit designs the new architecture.

**Interacts with:**
- **Plot Architecture audit:** This audit selects; Plot Architecture diagnoses. They form a selection → implementation → diagnosis cycle.
- **Series/Composite Novel audit:** For multi-work projects, selection must account for series-level architecture.
- **Fantasy & Series Architecture supplement:** For fantasy or other genre-specific series structures.
- **Genre modules:** Genre expectations constrain which spines are viable. Check genre module before finalizing selection.

**Output:** Structural plan (for new projects) or structural prescription (for stuck drafts), including spine selection, secondary engines, technique overlays, and risk assessment.
